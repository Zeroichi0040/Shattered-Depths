CHANGELOGS
v0.1.8 — Dynamic Combat
2026-03-09
Action Point System
Every entity now has an action point (AP) counter. Each turn cycle, entities gain AP equal to their rate (default 1). Actions are resolved in AP order — the entity with the most AP acts first, with the player winning ties.
This lays the foundation for characters and enemies with different speeds. A rate of 2 acts twice per cycle; a rate of 0.5 acts every other cycle. Rates can be decimals.
Telegraph & Bite
NEW Rats no longer deal instant damage when adjacent. Instead they enter a telegraph phase — targeting the tile the player is on and showing a pulsing red X with a red border.
NEW On the following turn the rat commits to biting that tile regardless of whether the player moved. Step off the marked tile to dodge; stay and take the hit.
Visual Feedback
NEW A white slash particle appears on any attacked tile — both when the player strikes an enemy and when a rat bites a tile.
NEW Waiting (SPACE on an empty tile) now shows a "..." float above the player.
v0.1.7 — Enemy Indicators & UI Overhaul
2026-03-08
Enemy Visuals
NEW Spawned enemies now have an amber HP bar, distinct from floor enemies (red). Makes it easy to tell at a glance which rats were always there and which appeared over time.
NEW Floor enemy counter in the dungeon header — shows how many enemies remain before the floor is cleared.
Save Data
NEW Export and import individual run slots. Share or back up a specific run without touching the others.
NEW Export and import all save data at once from the Progress screen. Useful for moving between devices or browsers.
UI
NEW Redesigned main menu with two-column layout — title and navigation on the left, animated ASCII dungeon preview on the right.
NEW Death screen split into two views: a compact run summary with a randomised death flavour message, and a full stats breakdown.
TWEAK Inventory UI updated. Character panel now shows class icon and player name at the top.
NEW Character name input on the stat allocation screen. Defaults to class name if left empty.
v0.1.6 — Enemy State Indicators
2026-03-08
State Indicators
  • ZZZ — enemy is sleeping. Three z's drift upward with staggered timing.
  • ! — enemy just woke up, surprised by the player.
  • ? — enemy is alert; lost sight of the player and is searching.
Pathfinding
TWEAK Improved enemy pathfinding. Rats no longer take large detours around each other unless the player is within line of sight — when hunting to a last-known position they walk through each other, preventing corridor gridlock.
v0.1.5 — Save Slots & Bug Fixes
2026-03-08
Save System
NEW Unlimited save slots. Previously capped; now as many slots as you want, each with per-slot import and export.
Bug Fixes
FIX XL floors with enemy respawn enabled can now be properly cleared. Spawned enemies no longer counted toward the kill requirement — only enemies that originated from floor generation do.
FIX Various minor UI and logic fixes.

v0.1.3 — Stats, Inventory & Enemy AI
2026-03-07
New Stats
  • DEF (Defense) — reduces incoming damage. Scales with CON at 0.02 DEF per point.
  • REGEN (Regeneration) — recovers HP each turn. Scales with CON at 0.001 REGEN per point.
Inventory
NEW Full inventory system: equip, unequip, drop, and pick up items. Base capacity is 10 slots; every 10 STR adds 1 more. Currently only starting weapons are in the game.
Enemy AI States
  • sleep — floor enemies are motionless until the player enters their FOV.
  • chase — actively pursues the player while in sight.
  • hunt — lost sight of player; walks to the last known tile.
  • alert — reached last known tile; waits 2–10 turns before roaming.
  • roam — wanders randomly. Once woken, enemies never return to sleep.
Spawned enemies start in roam state, never sleep.

QoL
Click a tile to auto-walk there. Click the player's tile to simulate SPACE (wait, interact, pick up). Bonfire glow now looks natural.
v0.1.2 — Auto Navigation
2026-03-06
Navigation Buttons
  • Ascend — auto-walks to the up-stairs tile and ascends.
  • Descend — auto-walks to the down-stairs tile and descends.
  • Bonfire — auto-walks to the nearest adjacent bonfire tile and heals.
If the player is already standing at the destination, the action fires immediately without walking.
Auto-walk Behaviour
  • Any key press cancels auto-walk.
  • Spotting an enemy mid-walk cancels auto-walk.
  • Walk speed significantly increased.
v0.1.1 — Level Scaling, Run Config & Enemy Spawning
2026-03-05
Level Scaling
Every level-up now increments all base stats (CON, STR, DEX, etc.) by 1 passively, in addition to any manually allocated points.
Run Config Screen
  • Difficulty — scales enemy stats and pacing.
  • Floor size — XS through XL. Upper floors are always fixed size regardless of this setting.
  • Enemy respawn — ON/OFF, defaulting based on difficulty. Can be overridden.
Floor Cleared System
  • Floor cleared banner and indicator when all floor enemies are dead.
  • Auto-navigation buttons appear on cleared floors.
  • No-respawn cleared floors: fog of war disabled, full map visible permanently.
  • Respawn-enabled cleared floors: layout stays explored but enemies may lurk in the dark.
Enemy Spawning
Enemies spawn on normal floors when respawn is enabled. Upper floors never spawn enemies. Spawn attempts occur every 25 turns on unseen tiles only.
Spawn probability uses exponential diminishing returns: 0.6^(n+1) where n is the number of currently alive enemies.
